# 정간보 생성기 — 기능·원리 정리

이 문서는 `index.html` 하나로 동작하는 정간보(井間譜) 생성기 웹앱이 제공하는 모든 기능과 그 동작 원리를 정리한 것입니다.

## 0. 개요

- **구조**: `index.html`(SVG 렌더 + 전체 로직) + `notes-data.js`(쉼표 이미지) + `symbols-data.js`(시김새 59종 SVG, `symbol_svgs/symbols` 원본) + `janggu-data.js`(장구 구음 6종 이미지). 이미지가 전부 base64로 파일 안에 내장되어 있어 외부 리소스 없이 동작합니다.
- **서버/DB 없음**: 정적 파일만으로 동작하는 클라이언트 사이드 앱이며 GitHub Pages로 배포됩니다.
- **자동 저장**: 브라우저 `localStorage`에 실시간 자동 저장되어 새로고침해도 유지됩니다. 별도로 `.jgb.json` 파일로 내보내기/불러오기도 가능합니다.
- **화면 구성**: 오른쪽(또는 상단) 사이드바에 **문서 / 레이아웃 / 출력** 3개 탭, 왼쪽/아래쪽에 악보 미리보기(SVG)와 편집창(선율 입력 + 필요 시 장단 입력)이 배치됩니다.

## 1. 데이터 모델 & 멜로디(선율) 문법

정간보의 구조는 **각(세로줄) → 정간(칸) → 분박(칸 안 세로 분할) → 음** 4단계로 이루어져 있고, 이는 텍스트로 아래와 같이 표현됩니다.

| 문법 | 의미 |
|---|---|
| 줄바꿈(⏎) | **각**(세로줄, 마디) 구분 |
| `\|` | **정간**(칸, 1박) 구분 |
| 정간 안 공백(스페이스) | **분박** 구분 — 그 칸 안에서 위→아래로 줄이 나뉨(2분박·3분박 등 박 등분) |
| 공백 없이 이어 쓰기 | **붙임** — 한 분박(행) 안에서 왼→오른쪽으로 가로 배치 |
| 한글 율명 | 황·대·태·협·고·중·유·임·이·남·무·응 (12율, 반음계 순서) |
| 옥타브 접두어 | 청(+1옥타브), 중청/겹청(+2), 배/탁(-1), 배탁/하배(-2). 접두어 없으면 중성(0) |
| `쉼` 또는 `쉼표` | 쉼표(무음) |
| `-` | 이음(연음, 앞 음을 지속) |
| `{이름}` | 시김새/기호 삽입. 예: `{흘림표}` |
| `{이름@크기%,좌우,상하}` | 시김새 개별 위치·크기 조정(자세한 건 3장) |

**렌더 규칙**: 각 안의 정간 개수는 실제로 입력된 `\|` 개수를 따르는 게 아니라 **문서 탭의 "정간 수" 설정(beats)을 항상 기준으로 그립니다** — 타이핑 중이라 `\|`가 덜 채워져도 격자가 끊기지 않고, 내용이 없는 칸은 빈 칸으로 표시됩니다.

**빈 각 처리**: `\|`로 나뉜 칸 수가 실제 선율 데이터에 없으면(각이 아예 없으면) 빈 격자를 그대로 유지해 다음 입력을 준비합니다.

## 2. 율명 표기 2가지 모드

사이드바(팔레트) 위 "표기" 드롭다운에서 선택합니다. (예전에 있던 "이미지" 모드는 제거되어 폰트 기반 2가지만 남음)

1. **폰트 (한자)**(기본값) — 황=黃, 태=太 같은 정식 한자로 표시. 옥타브가 있으면(청/배 등) **옥타브 변형 한자**(예: 청황=潢)를 우선 사용하고, 지원되는 변형자가 없으면 기본 한자 옆에 **옥타브 점**(●)을 위/아래로 옥타브 수만큼 찍어 표시합니다. 이때 변형자가 없어 점으로 대체된 글자는 옅은 회색(`#aaa`)으로 표시해 정식 변형자와 구분됩니다.
   - 변형 한자는 ±1옥타브, ±2옥타브까지만 매핑돼 있음(그 이상은 전부 점으로 표시).
   - 매핑은 `Cha-Unicode.docx`의 유니코드 표 기준으로 거의 전부 채워져 있음(+1옥타브 12개 음, -1옥타브 12개 음, +2옥타브 11개 음, -2옥타브 11개 음). 하배이·중청대 2개 조합만 변형자가 없어 점으로 대체(옅은 회색).
2. **폰트 (한글)** — 황, 청황, 하배협처럼 접두어+한글 그대로 표시. 2~3글자로 길어지는 만큼 글자 크기를 자동으로 줄여서(2글자 68%, 3글자 50% 등) 칸 밖으로 겹치지 않게 합니다. 팔레트에서도 동일하게 글자 수에 따라 자동 축소됩니다.

두 모드 모두 팔레트(음역×12율 매트릭스)와 실제 악보 렌더링에 동일하게 적용됩니다.

## 3. 시김새(장식음) — 59종

- 팔레트에서 "시김새" 탭을 누르면 **붙임표(음표 오른쪽에 작게, 예: 흘림표·농음표) / 독립 기호(한 칸을 통째로 차지, 예: 요성표) / 퇴성·추성(둘 다 가능)** 3그룹으로 나뉜 59종 시김새를 볼 수 있고 클릭하면 커서 위치에 삽입됩니다.
- 텍스트로는 `{한글이름}`으로 입력(예: `{흘림표}`).
- **개별 수정 모드**: 편집창 헤더의 "🎚 시김새 수정" 버튼을 누르면 켜지고, 악보에 그려진 시김새 기호를 클릭해 선택할 수 있습니다. 선택하면 우측 상단에 조정 패널이 뜨고 **크기(±10%p, 40~300% 범위) / 좌우(±5%p, ±90% 범위) / 상하(±5%p, ±90% 범위)** 버튼으로 조정하며, "위치·크기 초기화" 버튼으로 되돌릴 수 있습니다.
- 내부적으로는 `{이름@크기,좌우,상하}` 형태로 선율 텍스트에 저장됩니다(예: `{퇴성@150,10,-5}` = 150% 크기, 오른쪽으로 10%, 위로 5% 이동). 값이 기본값(100,0,0)이면 `@...` 부분 없이 순수 이름만 저장됩니다.
- 시김새 수정 모드가 켜져 있으면 정간 클릭 인라인 입력은 비활성화됩니다(서로 배타적).

## 4. 장단(장구) 입력

- **문서 탭 → "장단 추가"** 체크박스를 켜면 활성화됩니다.
- 켜면 편집창 하단에 "선율 입력"과 같은 구조의 **"장단 입력"** 창이 생기고, 궁·덩·기덕·덕·다·더러러러 6종 장구 구음 팔레트와 텍스트창이 나타납니다.
- 문법은 선율과 동일하게 `\|`(정간 구분)를 씁니다. 단, **장단은 반복되는 패턴이므로 곡 전체에서 맨 처음 각(가장 첫 세로줄) 옆에만 한 번 표시됩니다.** 정간 수는 선율의 정간 수(beats)와 자동으로 맞춰집니다.
- 렌더링 시 첫 각의 선율 칸 왼쪽에 얇은 별도 줄(선율보다 좁은 폭)로 장구 기호가 정간별로 그려지고, 그 뒤 각들은 그만큼 자연스럽게 왼쪽으로 당겨져 배치됩니다(전체 페이지 폭 계산에도 반영되어 넘치지 않음).
- 옵션을 끄면 장단 줄이 사라지고 정간이 원래 폭으로 돌아옵니다.

## 5. 문서 탭 (전체 설정)

| 항목 | 설명 |
|---|---|
| 종이 방향 | 세로(Portrait) / 가로(Landscape), A4 기준 |
| 한 각의 정간 수(beats) | 한 각(세로줄)에 들어갈 정간 개수 = 몇 박 장단인지 |
| 대강 분절 | 각 안에서 굵은 선으로 나눌 정간 묶음(예: `3 3 3 3`). 합이 정간 수와 같아야 하고, 다르면 무시된다는 경고가 뜸. 정간 수를 바꾸면 대표 패턴(8박=없음, 10박="7 3", 12박="3 3 3 3", 16박="3 2 3 3 2 3", 20박="6 4 4 6")이 자동으로 채워짐(직접 수정한 값은 덮어쓰지 않음) |
| 장단 추가 | 4장 참고 |
| 제목 / 부제 | 텍스트, 크기, 위치(mm) 각각 설정. 제목은 첫 페이지 첫 밴드의 오른쪽 칸에 세로로 크게, 부제는 그 아래 작게. 제목 글씨체(궁서체·고딕체·명조체·애플고딕·해서체) 선택 가능. 제목이 있으면 한 줄 각 수가 6 이상일 때 2개 각, 미만일 때 1개 각을 제목칸으로 씀 |

## 6. 레이아웃 탭 (크기·배치)

| 항목 | 설명 | 기본값 |
|---|---|---|
| 한 줄에 놓을 각 수 | 한 밴드(가로줄)에 나란히 놓을 각 개수 | 10 |
| 세로로 쌓을 밴드 수(최대) + 자동 | 한 페이지에 세로로 몇 줄 쌓을지. 자동이면 정간 수에 따라 1~3줄로 결정(12박 초과=1, 6박 이상=2, 그 미만=3) | 자동, 2 |
| 총 각 수(마디 수) + 자동 | 실제로 그릴 각 총 개수. 미입력 시 한 페이지를 꽉 채우는 값, 자동 체크 시 선율에 입력된 각 개수만큼만 | 18, 미체크 |
| 바깥 테두리 그리기 | 격자 전체를 감싸는 굵은 테두리 on/off | 켜짐 |
| 전체 배율 | 정간·간격을 한 번에 비례 확대/축소하는 슬라이더(0.6~2.5×) | 1.0× |
| 각 사이 세부 조정(펼치기) | 각 너비(mm), 각 사이 간격(mm), 밴드 사이 간격(mm)을 개별 지정 | 각 11mm / 간격 7mm / 밴드간격 10mm |
| 머리단(헤더) | 각 밴드 위에 이름 적을 빈 칸을 둘지 | 꺼짐 |

크기 설정은 A4 인쇄 영역을 넘으면 비율을 유지한 채 자동으로 축소됩니다.

## 7. 출력 탭

- **인쇄 / PDF 저장** — 브라우저 인쇄 기능 호출. 인쇄 시 사이드바·편집창·줌바·재생바 등 화면 전용 UI는 전부 숨겨지고 악보만 페이지별로 나뉘어 출력됩니다(가로/세로 방향에 맞춰 210×297mm 또는 297×210mm로 고정).
- **PNG 다운로드** — 각 페이지 SVG를 300dpi 기준(A4 세로 약 2480×3508px)으로 캔버스에 그려 PNG로 저장. 페이지가 여럿이면 파일명 뒤에 `-1`, `-2`… 붙음.
- **파일로 저장 / 불러오기** — 현재 문서 설정 전체(모든 필드 값 + 선율 + 장단 텍스트 + 각 수 자동 여부 등)를 `.jgb.json`으로 내보내거나, 그 파일을 다시 불러와 완전히 복원합니다. 곡을 따로 보관하거나 다른 사람과 공유할 때 씁니다.

## 8. 재생 (사인파 sonification)

줌 바 옆의 항상 보이는 **▶ 재생 / ■ 정지** 버튼(진한 색, 눈에 띄게 배치)과 **⚙** 설정 팝오버로 제공됩니다. 시김새·장단은 재생에 반영되지 않고 선율만 재생됩니다.

**설정(⚙ 팝오버)**
- 기준음(황·黃): **E♭4**(기본) 또는 **C4** 중 선택. 12율(황~응)은 여기서부터 반음계로 이어집니다.
- 템포(BPM): 1정간 = 1박 기준의 분당 박수.

**재생 규칙**
- 정간 1칸 = 1박.
- 칸 안 분박(공백으로 나뉜 행)은 그 박을 균등하게 나눔.
- **빈 정간과 이음(`-`)은 앞 음을 그대로 지속**(새 음을 시작하지 않고 직전 음의 길이만 늘림).
- **쉼표만 실제 무음** 처리(지속을 끊음).
- 시김새 단독 토큰이나 인식 안 되는 문자는 무시하되 소리는 끊기지 않게(지속) 처리.

재생 중에는 현재 재생 중인 정간이 악보 위에 파란색으로 하이라이트되며, 재생이 끝나면 자동으로 멈춥니다.

## 9. 편집 UI

- **정간 클릭 인라인 입력**: 악보의 빈 정간(또는 이미 입력된 정간)을 클릭하면 그 자리에 작은 입력창이 뜨고 바로 타이핑할 수 있습니다. `Enter`/`Tab`으로 확정 후 다음 정간으로 자동 이동(연속 입력 가능), `Escape`로 취소, 포커스를 잃으면(다른 곳 클릭) 확정만 되고 이동은 안 함.
- **팔레트 클릭 입력**: 율명/시김새/장단 팔레트의 기호를 클릭하면 편집창 커서 위치에 바로 삽입됩니다.
- **입력 방법 가이드(?)**: 선율 입력·장단 입력 각각의 헤더에 작은 물음표 버튼이 있고, 누르면 입력 방법(정간 클릭 vs 텍스트 입력)과 문법(줄바꿈·`\|`·분박 등) 설명이 뜹니다. 처음 방문했을 때 한 번 자동으로 열리고, 이후엔 버튼을 눌러야 다시 볼 수 있습니다.
- **줌**: 화면 확대(+)/축소(−)/**세로맞춤**(페이지 높이를 화면에 맞춤)/**가로맞춤**(페이지 너비를 화면에 맞춤). 화면 표시만 바뀔 뿐 실제 출력 크기와는 무관합니다.
- **되돌리기/초기화**: 선율·장단 각각 독립적인 되돌리기 스택을 가지고 있어, 초기화한 내용을 한 번 되돌릴 수 있습니다.
- **사이드바 접기**: 설정 폼 전체를 접어 화면을 넓게 쓸 수 있고, 우측 상단 "⚙︎ 설정" 버튼으로 다시 열 수 있습니다.

## 10. 자동 저장 / 파일 관리

- 거의 모든 입력이 바뀔 때마다(디바운스 없이) `localStorage`(`jgb_state_v1`)에 현재 상태 전체(모든 설정값 + 선율 + 장단 텍스트)가 즉시 저장됩니다. 새로고침해도 유지됩니다.
- 이는 "파일로 저장"(`.jgb.json` 내보내기)과는 별개입니다 — localStorage는 이 브라우저 안에서만 남는 자동 백업이고, 파일 저장은 곡을 다른 곳에 보관·공유하기 위한 명시적 내보내기입니다.

## 11. 알려진 제약 / 참고사항

- 프로젝트 폴더가 iCloud Drive 안에 있어, 일부 개발 환경(샌드박스)에서는 로컬 서버가 바로 뜨지 않을 수 있습니다(실제 사용자 컴퓨터에서는 문제없이 동작).
- 율명 이미지는 `notes-data.js`에 base64로 내장돼 있어 파일을 그냥 더블클릭해서 열어도(`file://`) 표시되지만, PNG 다운로드 시 이미지가 온전히 담기려면 로컬 서버로 열어야 하는 경우가 있습니다.
- `symbol_samples/` 원본 이미지 폴더는 용량 문제로 git에는 커밋되지 않고(`.gitignore`), 이미 필요한 것들은 `notes-data.js`/`symbols-data.js`/`janggu-data.js`에 base64로 내장돼 있습니다.
